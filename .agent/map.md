# Routing map — where things live

This is a routing table, not a description. It points; the detail lives in each
directory's `README.md`. It stays short and rarely goes stale because directory
locations change far less often than file contents. When you add or move a
top-level area, update this file in the same change.

| Area | Path | What it owns | Start here |
|------|------|--------------|------------|
| Monorepo | `packages/` + `apps/` | The layered uv-workspace — the whole system, one tree, one gate (ADR 0018). `packages/core` = `algotrading.core` (config/log/manifest/provenance — level 0). `packages/infra` = `algotrading.infra`: the **frozen contract seam** `infra/contracts` (table contracts, instrument key, `StorageRepository` protocol — M0 owns it) plus the full stack — the market-data plane (`connectivity`, `universe`, `collectors`), the pure-function analytics core (`pricing`, `snapshots`, `forwards`, `iv`, `surfaces`), the risk engine (`risk` — ADR 0006), QC + its sibling anomaly/triage plane (`qc` + `validation`, collapsed into one persisted `triage_records` table — ADR 0010), and the actor-driven operable layer (`orchestration` — jobs, qc_job, the five metrics, four alerts, dashboard, run-state ledger, EOD pipeline, the `reconstruction` subpackage — plus `observability` run-lineage over the `RunRegistry`). One driver, the actor, runs live and replay, which is what makes the headline acceptance tests (byte-identical replay, provenance, reconstruction, handover) pass inside the root gate. Nautilus is the runtime spine ([ADR 0023](decisions/0023-nautilus-runtime-spine-and-library-leverage.md)): IBKR rides Nautilus's adapter plus a custom Client-Portal REST transport (ADR 0024/0025); Saxo/Deribit are kept as `packages/infra-{saxo,deribit}` adapters; the collection seam is unified on the push `RawCollector` ([ADR 0027](decisions/0027-collection-seam-push-canonical.md)). `packages/infra-{ibkr,saxo,deribit}` = broker leaf adapters. `packages/{strategy,execution}` + `apps/frontend` (Python BFF + React/Vite web) = the upper layers. Layering enforced by import-linter; the only gate is the root `uv run ruff/mypy/lint-imports/pytest`. Each module's `README.md` is the next hop (mirrored under `documentation/modules/`). | `.agent/decisions/0018-monorepo-keystone-m0.md` |
| Operations | `documentation/` | Operator handover (E): the five runbooks (`documentation/runbooks/`), release-management rules + release notes (`documentation/releases/`), the frozen interface-contract list, and known limitations / support model. `documentation/modules/` mirrors every per-directory `README.md` via symlink, so each module's doc lives next to its code *and* under one roof here. Start at the index. | `documentation/README.md` |
| Blueprint | `documentation/blueprint/` | The founding domain reference: *Industrial Roadmap for a Volatility Infrastructure Platform v4.0*. Authoritative on domain (formulas, data contracts, field definitions, vocabulary) and the 16-step build order. Read `00-overview.md` first; Part II for math, Part III for the roadmap, Part IV for implementation guides. This is the plan of record (ADR 0011); `AGENTS.md` governs process, the blueprint governs domain — they do not conflict. | `documentation/blueprint/README.md` |
| Vol-surface pedagogy | `documentation/vol-surface/` | Pedagogical treatment of implied volatility and the vol surface: intuition, Greeks, smile anatomy, surface structure, no-arb checks. Rendered figures in `assets/`. Prerequisite reading before touching `infra/{iv,surfaces}`. | `documentation/vol-surface/vol_surface_pedagogique.md` |
| Notebooks | `notebooks/` | Demo pipelines for each broker (IBKR, Saxo, Deribit), pricing/Greeks demo, and surface-fit demo. Wire to the merged infra; honor as-of reproducibility rules — all data access must use the as-of abstraction, no wall-clock reads inside a notebook cell. | `notebooks/README.md` |
| Research | `research/` | Research notes and experiments. Reproducibility and as-of discipline rules. | `research/README.md` |
| Data | `data/` | Shared datasets (parquet/duckdb). Large/secret data stays out of git. | `data/README.md` |
| Tasks | `tasks/` | In-flight work claims (collision guard) and archived task notes. | `tasks/TASKBOARD.md` |
| Conventions | `.agent/conventions.md` | House style for code in any area. | `.agent/conventions.md` |
