# Routing map — where things live

This is a routing table, not a description. It points; the detail lives in each
directory's `README.md`. It stays short and rarely goes stale because directory
locations change far less often than file contents. When you add or move a
top-level area, update this file in the same change.

| Area | Path | What it owns | Start here |
|------|------|--------------|------------|
| Backend | `backend/` | Python service and quant logic (FastAPI, numpy/pandas/polars). Foundation (A): typed contracts, config/provenance, DuckDB-over-Parquet storage, fixtures, quality gate. Market-data plane (B): broker-agnostic connectivity, the instrument universe, and the append-only collector in `src/{connectivity,universe,collectors}` (each with a README). Analytics core (C): the pure-function quant heart — pricing engine, snapshot builder, forward/carry, IV solver, volatility surface in `src/{pricing,snapshots,forwards,iv,surfaces}` (each with a README). Risk engine (D): portfolio Greeks, monetized sensitivities, aggregation, broker reconciliation, and the scenario stress engine in `src/risk` (README + ADR 0006), built on C's frozen pricer. Integration & operations (E): the Nautilus actor that drives C/D and stamps their outputs in `src/actor` (under [ADR 0023](decisions/0023-nautilus-runtime-spine-and-library-leverage.md) Nautilus is the runtime spine; this `backend/` copy is the pre-Nautilus salvage source), the named-check QC library in `src/qc` plus its sibling anomaly/triage validation plane in `src/validation` (the two collapse into one persisted `triage_records` table — README + ADR 0010), and orchestration/observability + historical replay in `src/orchestration` (with the `reconstruction` subpackage) — README + ADR 0007. The same actor runs live and replay, which is what makes the headline byte-identical-replay and provenance-verification tests pass. | `backend/README.md` |
| Monorepo (merge target) | `packages/` + `apps/` | The layered uv-workspace the merge is building toward (ADR 0018). `packages/core` = `algotrading.core` (config/log/manifest/provenance — level 0). `packages/infra` = `algotrading.infra`: the **frozen contract seam** `infra/contracts` (table contracts, instrument key, `StorageRepository` + `BrokerSession` protocols — M0 owns it) plus the analytics/storage/market-data subpackages filled by M1–M7. The actor-driven operable layer landed under C3: `infra/orchestration` (jobs, qc_job, the five metrics, four alerts, dashboard, run-state ledger, EOD pipeline, and the `reconstruction` subpackage) + `infra/observability` (run-lineage `run_job` over the M10 `RunRegistry`) — one driver, the actor; the headline acceptance tests (byte-identical replay, provenance, reconstruction, handover) run inside the root gate. Live collection stays an injected seam pending C1 ([ADR 0026](decisions/0026-orchestration-observability-reconciliation.md)). `packages/infra-{ibkr,saxo,deribit}` = broker leaf adapters (M5). `packages/{strategy,execution}` + `apps/frontend` = upper layers. Layering enforced by import-linter; gate is the root `uv run ruff/mypy/lint-imports/pytest`. Start at the keystone ADR, then each package's README. The broker/actor seam is being reset by [ADR 0023](decisions/0023-nautilus-runtime-spine-and-library-leverage.md) — Nautilus is the runtime spine; IBKR on Nautilus's adapter, Saxo/Deribit kept. | `.agent/decisions/0018-monorepo-keystone-m0.md` |
| Operations | `documentation/` | Operator handover (E): the five runbooks (`documentation/runbooks/`), release-management rules + release notes (`documentation/releases/`), the frozen interface-contract list, and known limitations / support model. `documentation/modules/` mirrors every per-directory `README.md` via symlink, so each module's doc lives next to its code *and* under one roof here. Start at the index. | `documentation/README.md` |
| Blueprint | `documentation/blueprint/` | The founding domain reference: *Industrial Roadmap for a Volatility Infrastructure Platform v4.0*. Authoritative on domain (formulas, data contracts, field definitions, vocabulary) and the 16-step build order. Read `00-overview.md` first; Part II for math, Part III for the roadmap, Part IV for implementation guides. This is the plan of record (ADR 0011); `AGENTS.md` governs process, the blueprint governs domain — they do not conflict. | `documentation/blueprint/README.md` |
| Vol-surface pedagogy | `documentation/vol-surface/` | Pedagogical treatment of implied volatility and the vol surface: intuition, Greeks, smile anatomy, surface structure, no-arb checks. Rendered figures in `assets/`. Prerequisite reading before touching `infra/{iv,surfaces}`. | `documentation/vol-surface/vol_surface_pedagogique.md` |
| Notebooks | `notebooks/` | Demo pipelines for each broker (IBKR, Saxo, Deribit), pricing/Greeks demo, and surface-fit demo. Wire to the merged infra; honor as-of reproducibility rules — all data access must use the as-of abstraction, no wall-clock reads inside a notebook cell. | `notebooks/README.md` |
| Frontend | `frontend/` | JS/Vite app. Not scaffolded yet. | `frontend/README.md` |
| Research | `research/` | Research notes and experiments. Reproducibility and as-of discipline rules. | `research/README.md` |
| Data | `data/` | Shared datasets (parquet/duckdb). Large/secret data stays out of git. | `data/README.md` |
| Tasks | `tasks/` | In-flight work claims (collision guard) and archived task notes. | `tasks/TASKBOARD.md` |
| Conventions | `.agent/conventions.md` | House style for code in any area. | `.agent/conventions.md` |
| Glossary | `.agent/glossary.md` | Domain vocabulary an agent won't infer. | `.agent/glossary.md` |
| Decisions | `.agent/decisions/` | Append-only ADRs for non-obvious choices. | `.agent/decisions/` |
