Use when reviewing a backtest, a signal-generation pipeline, a factor model, or any code that makes predictions conditioned on a point in time. Produces a list of specific violations with file, line, and suggested fix.

# Process

## 1. Identify the time axis

Locate the code's notion of "now": the as-of date in each iteration, the rebalance dates, the prediction timestamps. If there is no explicit as-of abstraction and the code just iterates over rows, that is the first finding; every data access should be parameterizable by an as-of date.

## 2. Check each data source

For every dataset the code consumes, ask:

- **Is it keyed on the right date?** Fundamental data (earnings, filings, balance sheet items) must be keyed on publication or availability date, never on the reporting period end. A 10-K for fiscal year ending Dec 31 is typically not public until February or March; using Dec 31 as the key leaks two months of information.
- **Is it point-in-time?** Restated or revised data (GDP revisions, restated earnings) must be the vintage that was known at the as-of date, not the final value.
- **Are corporate actions and survivorship handled?** A universe constructed from currently-listed companies silently excludes failed ones.

## 3. Check rolling and expanding windows

For each rolling or expanding computation:

- Does the window at date `t` include observation `t` itself? For a signal used to trade on day `t`, this is look-ahead unless explicitly justified (e.g., the observation is a published close used for next-day trading and this is documented).
- Are overlapping returns being used without the correct variance adjustment?
- Is the window anchored to calendar dates (which can include future data during the holdout) or to index positions (which can silently shift meaning under resampling)?

## 4. Check the train/test split

- **No global preprocessing before the split.** Means, variances, PCA loadings, standardization parameters, and feature selection must be fit on train-only data and applied to test. A single `StandardScaler().fit_transform(X)` called before splitting is a violation.
- **Time-ordered splits only.** No random splits, no k-fold on time-series. Walk-forward with date-based splits; each test fold is strictly after its training data.
- **Test set touched once.** If the code re-tunes hyperparameters after looking at test performance, the test set is now contaminated.

## 5. Check the trading/execution layer

- Signals computed from close-of-day data used to trade at the same day's close is look-ahead; realistic assumption is next-day open or VWAP.
- Transaction costs applied? Slippage? If not, the reported returns are upper bounds, not estimates.
- Capacity constraints (ADV limits, position sizing relative to float) respected?

## 6. Check the robustness layer

For any headline result, confirm robustness checks exist:

- Subperiod stability (does the effect hold in the first half and the second half?).
- Universe perturbation (drop top/bottom decile by market cap, does it survive?).
- Parameter perturbation (±20% on window lengths, thresholds, smoothing; does the conclusion change?).
- Transaction cost sensitivity (at 10, 25, 50 bps, does the strategy still make sense?).

## 7. Check the trial count

Count how many variants have been tested in this research track (different signals, windows, universes, parameters). If the count exceeds five, flag it: multiple-testing correction is required, or the reported p-values are meaningless. Ask for the log of all trials run, including failures.

## 8. Reproducibility

Every number in a paper or report traces to a specific line of code. Every figure is generated by a script, not manually assembled. The full pipeline runs from one command on a fresh environment. If any of these fail, flag it.

# Output

Per finding: `path:line — <issue> (category: as-of / split / execution / trials / robustness) → <fix>`. If you cannot tell from the code alone whether a violation exists (e.g., you cannot see where the data was originally sourced), say so explicitly and name what information you need to decide.
