# research

Notebooks and experiments. The workspace README promised this directory; it now
exists.

## TL;DR

Exploratory work lives here. The rules below exist because research code has a
way of quietly becoming production assumptions, and because financial backtests
fail silently when discipline slips.

## Rules

- **Reproducibility.** Every number that ends up in a report or paper traces to a
  specific line of code. Every figure is generated by a script, not assembled by
  hand. The pipeline runs from one command on a fresh environment.
- **As-of discipline.** Any backtest, signal, or prediction reads data as it was
  known at the simulated decision time. All data access goes through an as-of
  abstraction. Fundamental data is keyed on publication date, not period end.
  See the `check-lookahead-bias` skill.
- **No global preprocessing before the train/test split.** Walk-forward,
  date-based splits. Touch the test set once.
- **Log every variant**, including failures. Past five trials, correct for
  multiple testing — the reported p-values are otherwise meaningless.
- **Notebook → script.** A result worth keeping gets promoted from a notebook to
  a script under version control. Notebooks are for exploring, not for being the
  source of truth.

## Conventions

Follows `/srv/project/.agent/conventions.md`. Use `uv` for environments and
dependencies. Keep large or secret datasets in `data/` (and out of git), not here.
