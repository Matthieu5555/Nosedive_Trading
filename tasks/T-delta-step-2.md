## Constat (the bug)

The prof asked for a **±30Δ band by steps of 2**:
`30/28/…/04/02/ATM/02/04/…/28/30` — puts −0.30…−0.02, ATM, calls +0.02…+0.30. The
projection instead delivered a **step-10, 8-point** band, and that band lived as a `.py`
literal (`_DEFAULT_BAND_LABELS`/`_TARGETS` in `surfaces/projection.py`), instantiated by the
driver with no config override (`ProjectionConfig(version=…)`), with **no YAML supplying the
band**. The economic intent (the grid the prof drew) was silently overwritten by a technical
default, and the gate stayed green because:

- the QC `delta_band_completeness` ran against `max_delta_step = 0.25` — a coarse grid passed;
- and it spanned the **realized** greek delta, where the ATM pillars sit at ≈ ±0.5, so a 0.02
  step across ATM could never be enforced anyway.

## Ce qui a changé

1. **Typed config home (ADR 0028).** The band is generated, never hand-listed:
   `delta_band_axis(band_low_delta, band_high_delta, band_step)` in `surfaces/projection.py`
   expands a signed band (validated, hundredths grid) into the `(labels, targets)` axis, and
   `ProjectionConfig.from_band(...)` builds the config from it. The `.py` module default is now
   *generated by the same expander* (no literal to drift), as the in-memory/test placeholder.
2. **`GridQcConfig.band_step`** (new field, `core/config/platform_config.py`) +
   `configs/qc.yaml` `grid.band_step: 0.02`. The driver
   (`actor/driver.py` `_build_projected_analytics`) now builds the axis from
   `config.qc_threshold.grid.{band_low_delta, band_high_delta, band_step}` via `from_band` —
   **zero `.py` literal at the call site**. `PROJECTION_AXES_VERSION` → `1.2.0`.
   One band definition feeds *both* the projection (emits) and the QC (validates) → no drift.
3. **`configs/qc.yaml` `max_delta_step` 0.25 → 0.02** so the QC actually *forces* the step: a
   dropped point opens a `0.04` hole (`2·band_step`) and fails.
4. **QC spans `target_delta`, not realized `delta`** (`qc/inputs.py` Protocol + `qc/checks.py`
   `check_delta_band_completeness`). The band is a *target*-delta axis; the ATM pillars sit at
   `0.0` there (their realized deltas are ≈ ±0.5), which is the only coordinate that fills the
   ATM centre and makes the pas-2 step enforceable. Test fixtures already used the band
   coordinate, so this aligns the check with them.
5. **`atm` + `atmp` kept** (owner ruling 2026-06-13): the front straddle template
   (`apps/frontend/web/.../basketTemplates.ts`) composes `long atm + long atmp`, and the
   blueprint/pedagogical reference treat the smile as ONE curve with two wings (put/call split
   is a *display* + a *greeks* concern → follow-up `T-mirror-greeks-putcall`). So the default is
   15 puts + atm + atmp + 15 calls = **32 cells** per spanned tenor.
